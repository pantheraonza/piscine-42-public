Rush 02 — Arquitectura & Mapa Mental

Archivos · Funciones · Lógicas · Decisiones de diseño · Makefile · Dependencias

./rush-02 [dict_path] número

Convierte un número entero sin signo en su valor escrito en inglés usando un diccionario externo. Acepta 1 argumento (solo el número) o 2 (diccionario alternativo + número). El programa habla en inglés. Nunca convierte el número a un tipo entero — opera siempre sobre strings, lo que le permite escalar a cualquier número que quepa en el diccionario sin límite de tamaño. Diseñado con Divide & Conquer: procesos independientes desarrollados y verificados por separado.

C · Norma 42 · 0 memory leaks
Decisiones de diseño

8 decisiones que hacen funcionar la solución por qué funciona

Decisión 1 — Estrategia

Divide para conquistar: procesos, funciones clave y aceleradores

El problema se divide en procesos independientes: validar el input, leer el diccionario, parsear las líneas, convertir el número. Cada proceso tiene funciones clave que hacen el trabajo central y funciones de soporte (ft_strlen, ft_strcpy, ft_strrev, ft_trim_zeros) que aceleran y simplifican las demás. Ninguna función supera 25 líneas ni 5 variables — Norma 42.

Decisión 2 — Parseo

Parseo del diccionario inspirado en atoi

El subject dice: "los números deben manejarse igual que atoi". split_line aplica esa lógica: salta espacios al principio, lee dígitos hasta el ':', salta el separador y los espacios, copia el valor hasta fin de línea. La clave extraída se guarda siempre como string — nunca se convierte a entero.

Decisión 3 — Estructura de datos

struct t_dict como tabla de consulta

Un struct agrupa los dos campos de cada entrada: dict_number (la clave numérica como string) y dict_words (la palabra). Un array de 50 structs en main() es la tabla completa del diccionario. Norma 42 exige prefijo s_ para la estructura y t_ para el typedef.

Decisión 4 — Búsqueda

Best match para encontrar la clave correcta

find_dict_key recorre todo el diccionario guardando siempre la clave más grande que sea ≤ N. No depende del orden de las entradas. Funciona con diccionarios desordenados, con líneas vacías y con claves extra añadidas — siempre encuentra la clave canónica correcta.

Decisión 5 — Algoritmo

Recursión para descomponer cualquier número

ft_convert_input se llama a sí misma para los multiplicadores. Para 2542: divide entre 1000 → cociente "2" → recursión → imprime "two" → imprime "thousand" → continúa con "542". La recursión maneja cualquier profundidad sin lógica adicional, desde "forty two" hasta "nine hundred ninety nine trillion".

Decisión 6 — Dos estrategias

Sumar para ≤2 dígitos, multiplicar para ≥3 dígitos

Claves de 1-2 dígitos (20, 9, 40…) son aditivas: se restan de N y se imprime su palabra. Claves de 3+ dígitos (100, 1000, 1000000…) son multiplicadoras: se divide N entre la clave, se convierte recursivamente el cociente, se imprime la palabra, y se continúa con el resto. La longitud de la clave determina la estrategia.

Decisión 7 — Corrección

ft_trim_zeros: cortar los ceros que sobran

Después de cada resta, ft_strsub puede dejar ceros a la izquierda: "42 − 40 = 02". Pero "02" y "2" son distintos como strings — ft_strcmp los trataría diferente y find_dict_key nunca encontraría la clave "2". ft_trim_zeros limpia esto en cada paso. Sin esta función el programa entraría en bucle infinito.

Decisión 8 — Memoria

Limpiar la memoria a lo largo del proceso

malloc se usa 3 veces por entrada del diccionario (1 buffer + 1 dict_number + 1 dict_words) → hasta 83 mallocs para 41 entradas.
free se usa en 3 momentos:
· Si open falla → free(buffer) inmediato en read_file
· Tras parsear todas las líneas → free(buffer) en read_diccionary
· Al terminar el programa → free_dict() libera los 82 campos de las 41 entradas
Verificado con valgrind: 0 leaks.

Funciones autorizadas por el subject — cómo las usamos

write
Única función de salida. Imprime palabras, espacios, el '\n' final y los mensajes de error. Se usa en ft_convert_add, ft_convert_mul, y en main para errores.
malloc
Reserva memoria para el buffer del archivo (8193 bytes) y para cada dict_number (64 bytes) y dict_words (256 bytes) de cada entrada. Hasta 83 llamadas para 41 entradas.
free
Libera en 3 momentos: si open falla (buffer), tras parsear (buffer), y al final (free_dict libera los 82 campos de las 41 entradas). Cada malloc tiene su free.
open
Abre el .dict en modo solo lectura con O_RDONLY (de fcntl.h). Devuelve un file descriptor — número entero que identifica el archivo abierto en el sistema operativo.
read
Lee hasta 8192 bytes del archivo y los guarda en el buffer. Devuelve cuántos bytes leyó realmente. El '\0' hay que añadirlo manualmente: buffer[bytes] = '\0'.
close
Cierra el archivo tras leerlo. Obligatorio para no agotar los file descriptors del sistema operativo. Siempre se cierra lo que se abre.
struct + typedef
Parte del lenguaje C, no librería. Agrupa dict_number y dict_words en t_dict. Norma 42: prefijo s_ para struct, t_ para typedef. Array de 50 structs = tabla completa.
Makefile
Sistema de construcción obligatorio por el subject. Define cómo compilar con -Wall -Wextra -Werror. Reglas requeridas: all, clean, fclean, re. Regla extra: sani para debugging.
Flujo de ejecución
El camino desde el terminal hasta la respuesta
1. main()
Punto de entrada. Decide qué diccionario usar (argv[1] si hay 3 args, "numbers.dict" si no). Llama a todo en orden y escribe '\n' al final.
→
2. validate_input()
¿Es un número entero sin signo? Comprueba cada char entre '0' y '9'. Si falla → write "Error\n" y el programa termina.
→
3. read_diccionary()
Coordina la lectura: llama read_file para cargar el archivo y fill_dictionary para procesarlo línea a línea. Devuelve cuántas entradas tiene el diccionario.
→
4. fill_struct() ×N
Por cada línea válida: split_line separa clave y valor en el ':', fill_struct los guarda en t_dict[] con malloc. Construye la tabla de consulta.
→
5. ft_convert_input()
Motor de conversión. While N>"0": find_dict_key busca el best match → si clave ≥3 dígitos → ft_convert_mul (recursivo) → si clave ≤2 dígitos → ft_convert_add.
→
6. write("\n")
Añade el salto de línea final requerido por el subject.
→
7. free_dict()
Libera toda la memoria del diccionario: dict_number y dict_words de cada entrada. Verificado con valgrind: 0 leaks.
Salidas de error: → Error\n: input no es número entero sin signo (letras, decimales, negativos, vacío)  |  → Dict Error\n: el archivo .dict no se puede abrir, tiene formato incorrecto, o no tiene claves suficientes para resolver el número
Archivos y funciones

validate_input.c

ft_strlen(char *str)
Cuenta cuántos caracteres hay hasta el '\0'. Implementación propia — strlen está prohibida. Función de soporte usada por casi todas las demás.
validate_input(argc, argv)
Puerta de entrada. Verifica que hay 2-3 argumentos y que el último tiene solo dígitos '0'-'9'. Rechaza puntos, letras y negativos. Devuelve 1 si válido, 0 si no.
Usada por: main.c

parse_dictionary.c

ft_strcpy(dst, src)
Copia un string completo a otro. Función de soporte para guardar clave y valor en la memoria reservada con malloc.
split_line(line, dict_number, dict_words)
Separa "42 : forty two" en "42" y "forty two". Inspirada en atoi: salta espacios, lee dígitos hasta ':', salta separadores, copia el valor. Función clave del parseo.
fill_struct(line, dict_entry)
Llama split_line y pide memoria con malloc para guardar el par clave-valor en la tabla t_dict. Construye cada fila de la tabla de consulta.
Usada por: read_dictionary.c

read_dictionary.c — coordina la lectura del archivo

read_file(file_path)
Abre el archivo con open (O_RDONLY), pide 8193 bytes con malloc, lee con read, cierra con close. Añade '\0' manual al final. Devuelve el contenido completo como string. malloc #1
create_line(file, line, i)
Extrae una sola línea del buffer desde la posición i hasta '\n'. Devuelve su longitud, lo que permite avanzar correctamente en el siguiente paso.
process_entry(file, line, dict, count)
Recibe una línea y llama fill_struct para guardarla. Si fill_struct falla, libera el buffer (free #1) y devuelve error para que el proceso se detenga limpiamente.
fill_dictionary(file, dict)
Recorre el buffer llamando create_line línea a línea. Salta líneas vacías (j==0). Llama process_entry para cada línea válida. Devuelve el total de entradas guardadas.
read_diccionary(file_path, dict) — función principal
Orquesta todo: read_file → fill_dictionary → free(buffer) (free #2). Devuelve el número de entradas o -1 si algo falla.
Usada por: main.c | Usa: parse_dictionary.c

rush-02.h — el contrato del programa

typedef struct s_dict → t_dict
Define la "ficha" de cada entrada: dict_number (clave como string, ej. "1000") y dict_words (palabra, ej. "thousand"). El array t_dict dict[50] en main() es la tabla completa. Norma 42: s_ para struct, t_ para typedef.
17 prototipos de funciones
Un prototipo es la firma de una función sin su cuerpo. Permite que cualquier archivo .c llame a funciones definidas en otros archivos. Sin el header, el compilador no sabe que existen. Es el índice del programa.
#include unistd.h · fcntl.h · stdlib.h
unistd.h habilita write, read, close. fcntl.h habilita open y O_RDONLY. stdlib.h habilita malloc y free. Al incluir solo rush-02.h en cada .c, todos tienen acceso a todo automáticamente.
Incluido por: todos los archivos .c

string_math.c — aritmética sobre strings

⚠ Pareado con string_mathtwo.c — la Norma 42 limita a 25 líneas por función y 5 funciones por archivo. El mismo bloque funcional se divide entre dos archivos.


ft_strrev(str) — soporte
Invierte un string in-place con dos índices. Necesaria porque ft_count_to_str construye los dígitos del número al revés y hay que darles la vuelta.
ft_strcmp(a, b) — clave
Compara dos números-string: longitud primero (el más largo es mayor), luego char a char si tienen igual longitud. Base del best match y de todas las decisiones de conversión.
ft_strsub(result, a, b) — clave
Resta dos números-string dígito a dígito desde la derecha con carry, igual que en papel. Para la estrategia aditiva (claves ≤2 dígitos) y para calcular el resto en la división.
ft_strdiv(result, remainder, a, b) — clave
Divide por restas sucesivas: resta b de a hasta que a sea menor que b, contando las veces. El contador es el cociente (va a result), lo que queda es el resto (va a remainder). Para la estrategia multiplicadora.
ft_trim_zeros(number) — soporte crítico
Elimina ceros a la izquierda: "042"→"42". Se llama después de cada operación aritmética. Sin esta función ft_strcmp falla y el programa entra en bucle infinito. Es el pegamento que hace funcionar todo el sistema.
Usada por: string_mathtwo.c y convert_input.c

string_mathtwo.c — helpers de conversión

⚠ Pareado con string_math.c — mismo bloque funcional dividido por límite de la Norma 42.


ft_count_to_str(result, count) — soporte
Convierte el cociente entero (resultado de la división) de vuelta a string. Extrae dígitos con % 10 en orden inverso, luego usa ft_strrev para darle la vuelta. Permite que el cociente siga procesándose como string en la recursión.
ft_convert_mul(number, dict, dict_size, idx) — clave
Estrategia multiplicadora para claves ≥3 dígitos (thousand, million…). Divide number entre la clave → obtiene cociente y resto → llama recursivamente ft_convert_input(cociente) → escribe la palabra de la clave → continúa con el resto.
ft_convert_add(number, dict, idx) — clave
Estrategia aditiva para claves ≤2 dígitos (twenty, nine…). Escribe la palabra, resta la clave de number con ft_strsub, y hace trim. Simple y directo.
Usada por: convert_input.c | Usa: string_math.c

convert_input.c — el algoritmo central

find_dict_key(dict, dict_size, number) — BEST MATCH
Recorre todo el diccionario de principio a fin. Para cada entrada ≤ N compara si es mayor que el mejor candidato actual. Al terminar devuelve el índice de la clave más grande que cabe en N. No necesita que el diccionario esté ordenado.
ft_convert_input(number, dict, dict_size) — motor
Caso especial N=="0" → escribe "zero". While N>"0": find_dict_key busca el best match → si longitud de clave > 2 → ft_convert_mul (recursivo para el cociente) → si no → ft_convert_add. Se llama recursivamente desde ft_convert_mul.
Usada por: main.c | Llamada recursivamente desde: string_mathtwo.c (ft_convert_mul)

main.c — el orquestador

free_dict(dict, size) — limpieza final
Libera la memoria de cada entrada: free(dict[i].dict_number) y free(dict[i].dict_words) para i=0 a size-1. Son los 82 free que corresponden a los 82 malloc de fill_struct para 41 entradas. free #3 en el proceso global.
main(argc, argv) — orquestador
validate_input → si falla: write "Error\n", return 1 → elige dict_path → read_diccionary → si falla: write "Dict Error\n", return 1 → ft_convert_input → write "\n" → free_dict → return 0.
Orquestador principal. Usa todos los archivos vía rush-02.h
Makefile — sistema de construcción

¿Para qué sirve el Makefile y cómo está construido?

El Makefile es el sistema de construcción del proyecto. Le dice al compilador qué compilar, con qué opciones y en qué orden. Sin él el evaluador de 42 no puede compilar el proyecto. El subject exige obligatoriamente las reglas $NAME, clean y fclean.

# Nombre del ejecutable NAME = rush-02 CC = cc CFLAGS = -Wall -Wextra -Werror # Archivos fuente SRCS = main.c validate_input.c \ parse_dictionary.c \ read_dictionary.c \ string_math.c string_mathtwo.c \ convert_input.c OBJS = $(SRCS:.c=.o) all: $(NAME) $(NAME): $(OBJS) $(CC) $(CFLAGS) $(OBJS) -o $(NAME) %.o: %.c $(CC) $(CFLAGS) -c $< -o $@ # Solo desarrollo — no entregar sani: $(CC) $(CFLAGS) $(SRCS) \ -fsanitize=address -o $(NAME) clean: rm -f $(OBJS) fclean: clean rm -f $(NAME) re: fclean all .PHONY: all clean fclean re sani
make  /  make all
Compila cada .c en un .o y luego los enlaza en rush-02. Las flags -Wall -Wextra -Werror convierten cualquier warning en error — código siempre limpio.
make sani
Compila con AddressSanitizer. Detecta errores de memoria en tiempo real durante desarrollo. Nunca se entrega con esta opción activada.
make clean
Elimina los archivos .o intermedios. El ejecutable rush-02 se conserva.
make fclean
Elimina todo: .o y el ejecutable. El directorio vuelve al estado inicial. Requerido por el subject.
make re
fclean + all. Recompila desde cero. Útil cuando cambias rush-02.h o quieres asegurarte de no tener objetos obsoletos.
.PHONY
Declara que all, clean, fclean, re, sani son nombres de reglas y no archivos. Sin esto, si existiera un archivo llamado "clean", make lo ignoraría.
